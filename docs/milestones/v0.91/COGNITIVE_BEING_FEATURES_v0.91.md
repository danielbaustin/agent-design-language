# v0.91 Cognitive-Being Feature Allocation

## Status

First-pass planning allocation for v0.91 and the v0.91.1 adjacent-systems
follow-on.
This document does not create the final issue wave.

v0.91 remains downstream of Runtime v2 and upstream of the v0.92 identity and
first-birthday milestone. It should make the cognitive-being prerequisites
reviewable without claiming birth, consciousness, legal personhood, or final
constitutional citizenship.

## Purpose

The v0.91 planning corpus now contains more than moral trace. It contains the
first explicit ADL surfaces for wellbeing, kindness, humor and absurdity,
affect-like reasoning control, cultivated intelligence, moral resources, and
secure agent communication.

The planning goal is to keep those as first-class engineering work rather than
letting them remain philosophy notes or pulling them into the wrong milestone.

## Runtime And Identity Boundary

Runtime v2 is already an inherited substrate. v0.91 should consume Runtime v2,
citizen-state, Freedom Gate, trace, standing, access, projection, and governed
tool surfaces where needed. It should not reopen Runtime v2 foundations as new
v0.91 work unless a specific prerequisite gap is discovered.

Identity, continuity, memory grounding, and the first true birthday remain
v0.92. v0.91 may produce evidence that v0.92 needs, but it should not implement
the birthday boundary or claim durable identity architecture as complete.

## First-Class Feature Set

| Feature surface | v0.91 role | Expected evidence |
| --- | --- | --- |
| Wellbeing and happiness | Decomposable diagnostic tied to coherence, agency, continuity, progress, moral integrity, and participation. | Diagnostic contract, self-access and redacted-view examples, negative cases against scalar happiness. |
| Kindness | Inspectable non-harm, dignity, autonomy, constructive benefit, and long-horizon support behavior. | Kindness contract, conflict fixtures, refusal/support examples, review packet. |
| Humor and absurdity | Bounded frame-detection and reframing surface, not entertainment theater. | Absurdity/reframing event, fixtures for contradiction or wrong-frame detection, safety constraints. |
| Affect / emotional model | Explicit reasoning-control signals such as confidence, tension, curiosity, caution, frustration, satisfaction, escalation, and memory priority. | Affect-signal schema or report, policy hooks, negative cases against hidden emotion claims. |
| Cultivating intelligence | Formation layer that turns raw capability into bounded, civil, reality-grounded, morally participating intelligence. | Formation contract, trace links, review criteria for restraint, reasonableness, and learning posture. |
| Moral resources | Care, refusal, anti-dehumanization, moral attention, persistence, and other-recognition as durable design resources. | Moral-resource record, fixtures, review criteria, trace linkage. |
| Moral governance | Moral events, validation, moral trace, outcome linkage, metrics, trajectory review, and anti-harm constraints. | Existing v0.91 moral-governance evidence surfaces. |
| Agent communication prerequisites | Secure, identity-bound, traceable intra-polis communication and invocation before broader social cognition. | ACIP split plan, secure envelope/invocation records, redaction and trace examples. |

## v0.91 Versus v0.91.1 Split

v0.91 should carry the core cognitive-being substrate:

- moral event, validation, trace, outcome, metrics, trajectory, and anti-harm
  foundations
- wellbeing diagnostic v0
- moral resources v0
- first concrete kindness and affect/reframing slices
- secure intra-polis ACIP prerequisites needed by review and demos
- a flagship cognitive-being proof demo

v0.91.1 is not just generic overflow. It is the follow-on planning and
architecture milestone for adjacent systems that are too large to absorb into
the v0.91 core:

- capability and aptitude testing
- intelligence metric architecture
- ANRM / Gemma and local-model evidence architecture
- Theory of Mind planning
- memory and identity source-doc alignment for the v0.92 birthday lane
- runtime-v2 and polis documentation alignment
- ACIP conformance, redaction, and local-security hardening if not finished in
  v0.91
- review-finding remediation from v0.91 where it affects those adjacent systems

## Acceptance Bar

Each first-class cognitive-being feature should have:

- a named contract, schema, record, or review surface
- positive and negative fixtures where applicable
- trace and evidence linkage
- visibility and privacy policy where citizen or operator views differ
- demo or proof coverage, or an explicit non-proving deferral
- clear non-claims

## Non-Claims

v0.91 does not claim solved morality, solved happiness, production moral
agency, consciousness, legal personhood, constitutional citizenship, a scalar
karma score, a scalar happiness score, or the first true birthday.

v0.91 also does not claim complete identity architecture. It prepares evidence
that v0.92 can use when identity, continuity, memory grounding, and birthday
semantics become primary.
