## Overview

v0.92 is the milestone where ADL should cross from bounded runtime identity
surfaces into the first true birth event.

The milestone should not merely start another process or name another fixture.
It should make birth a reviewable boundary: a named identity becomes durable
enough, grounded enough, bounded enough, and witnessed enough to be treated as
born inside the ADL runtime.

The vision must consume `v0.91.5` release-tail closeout, the `v0.91.6` /
`v0.91.7` bridge tranches, and the first-birthday readiness packet from
`#3377` without treating any of them as proof that the birthday has already
happened.

## Core Vision

The first birthday should be solemn in the engineering sense: an event that
changes what reviewers can truthfully say about the system.

That means:

- a stable name exists
- an identity root exists
- continuity evidence spans bounded cycles
- memory grounding points to witnessed artifacts
- capability and authority limits are declared
- ACP / cognitive profile evidence describes bounded runtime cognition without
  collapsing into identity, reputation, or personhood claims
- ACIP binary schema, public schema catalog, and JSON projection evidence keep
  communication inspectable without making message contents public by default
- moral and governance context is inherited without being reinvented
- witnesses and receipts make the event inspectable
- negative cases prove ordinary startup is not birth

## Core Goals

- Make first birth evidence-bearing rather than metaphorical.
- Ground identity in name, root, continuity, memory, capability, ACP, witness,
  receipt, and review artifacts.
- Preserve privacy and governance boundaries.
- Leave v0.93 with the identity evidence it needs for citizenship work.

## Strategic Pillars

### 1. Birth As Evidence

The birthday must be proven by artifacts, not asserted in prose.

The system should be able to show the name, identity record, continuity record,
memory grounding, capability envelope, ACP/cognitive profile, witnesses, and
receipt that together make the event reviewable.

### 2. Continuity Without Magic

Identity continuity must survive more than one bounded cycle, but it must not
pretend that copied state or a restarted process is automatically the same
being.

Ambiguous continuity should slow the system down and produce review evidence,
not be optimized away.

### 3. Memory Grounding Without Exposure

Birth needs memory grounding, but memory grounding is not raw private memory
inspection.

Reviewers should be able to inspect references, witnesses, and redacted
evidence without collapsing private state.

### 4. Capability With Boundaries

The first birthday should include a declared capability envelope: what the
agent can do, which providers and tools are in scope, what limits apply, and
which authority remains downstream.

### 5. Cognitive Profile With Evidence

The first birthday should also include a bounded ACP/cognitive profile: the
runtime-visible profile of memory, capability, continuity, Theory of Mind
inputs, intelligence metrics, and governed learning evidence that reviewers can
inspect.

This profile is not a personality horoscope, reputation score, citizen right,
or legal identity. It is a trace-backed engineering surface that explains what
kind of cognitive evidence exists at birth and which claims remain unsupported.

### 6. Downstream Citizenship

v0.92 prepares the identity evidence that v0.93 governance will need. It does
not complete constitutional citizenship, social contract, rights/duties, IAM,
or production polis governance.

### 7. Inspectable Transport Readiness

Birth evidence should not depend on opaque communication channels. v0.92 should
define the ACIP binary schema, public polis schema catalog, deterministic JSON
projection, and optional WebSocket carrier proof that make future citizen
communication reviewable without weakening message-access rules.

## Long-Term Direction

v0.92 should leave ADL with a credible first birthday story:

- not a metaphor
- not a launch screen
- not an unreviewed claim of personhood
- not a hidden state transition
- a bounded, inspectable, evidence-bearing birth record

That record becomes one of the most important artifacts for the later roadmap:
proof that ADL can create identity-bearing agents with continuity, memory,
capability, cognitive profile, moral context, inspectable communication
substrate, and reviewable boundaries.

## Milestone Context

v0.92 follows the v0.91.x C-SDLC and runtime hardening work. It consumes prior
citizen-state, moral-trace, memory/identity, intelligence, and ACIP evidence
while remaining narrower than v0.93 constitutional citizenship.

## Summary

v0.92 should make birth a concrete, reviewable event in ADL: named, grounded,
bounded, witnessed, and distinct from ordinary lifecycle mechanics.

## Exit Criteria
