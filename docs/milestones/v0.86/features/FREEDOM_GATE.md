# Freedom Gate

## Purpose

Define the **Freedom Gate** as a core architectural mechanism in ADL that enables bounded autonomy, moral agency, and accountable action selection.

The Freedom Gate is the point at which an agent transitions from:
- reasoning → action  
- evaluation → commitment  
- possibility → choice  

It is not a UI concept, nor a simple approval step.  
It is the **structural locus of agency**.

---

## Overview

All meaningful agents must eventually **choose**.

Reasoning alone is not agency.  
Planning alone is not agency.  
Simulation alone is not agency.

Agency requires:
- the ability to **select among alternatives**
- the willingness to **commit to a course of action**
- the acceptance of **consequences over time**

The Freedom Gate is where this occurs.

> **The Freedom Gate is the architectural boundary between cognition and action.**

It is the moment where:
- multiple candidate actions exist
- constraints, values, and context are evaluated
- one action is selected and enacted

---

## Core Claim

> **Without a Freedom Gate, there is no agency—only execution.**

A system that:
- always follows instructions
- never evaluates competing options
- cannot refuse or defer

is not an agent.

It is a tool.

The Freedom Gate is what transforms a system from:
- reactive → deliberative  
- stateless → accountable  
- obedient → responsible  

---

## How It Works

### 1. Inputs to the Gate

The Freedom Gate operates on a structured set of inputs:

- **Candidate actions**
  - generated by reasoning, planning, or instinct
- **Context**
  - current state, environment, constraints
- **Identity**
  - commitments, roles, narrative continuity
- **Affect**
  - urgency, importance, emotional weighting
- **Policies / norms**
  - rules, constraints, learned preferences
- **Chronosense**
  - temporal awareness (recency, delay, deadlines)

These inputs define the **decision surface**.

---

### 2. Evaluation Phase

Before selection, the agent evaluates:

- feasibility
- consequences
- alignment with commitments
- temporal implications
- moral / normative considerations

This phase may involve:
- reasoning graphs
- simulation
- policy lookup
- affect weighting

Importantly:

> Evaluation does not yet produce action.

---

### 3. Commitment (The Gate Itself)

The Freedom Gate is crossed when:

- one option is selected
- alternatives are rejected (explicitly or implicitly)
- the agent **commits**

This commitment has properties:

- **irreversibility (partial or full)**
- **traceability**
- **accountability**

At this point:
- the agent is no longer exploring
- it is acting

---

### 4. Output

The result of the Freedom Gate is:

- a chosen action
- a justification (implicit or explicit)
- a traceable decision event

This event should be:
- logged
- reviewable
- attributable to the agent

For the implemented `v0.86` bounded runtime surface, the primary emitted proof surface is `freedom_gate.v1.json`, and the milestone demo proof surfaces are:

- `artifacts/v086/freedom_gate/allowed_case.json`
- `artifacts/v086/freedom_gate/blocked_case.json`
- `artifacts/v086/freedom_gate/summary.txt`

## Architectural Role

The Freedom Gate sits at the center of several systems:

### Chronosense
- provides temporal grounding
- enables delay, urgency, and sequencing to matter

### Identity
- ensures decisions are consistent over time
- binds choices into a narrative self

### Affect
- applies pressure and prioritization
- influences urgency and importance

### Reputation (future)
- records patterns of decisions
- enables trust and evaluation

### Signed Trace
- captures decision events
- enables verification and audit

> The Freedom Gate is the hinge that connects all of these.

---

## Why It Matters

Without a Freedom Gate:

- there is no real refusal
- there is no meaningful prioritization
- there is no ownership of action
- there is no moral responsibility

The system becomes:

- a pipeline
- a function
- a responder

With a Freedom Gate:

- actions are chosen, not merely triggered
- commitments persist over time
- decisions can be evaluated and judged
- behavior becomes **character-like**

---

## Moral Dimension

The Freedom Gate is where **moral agency emerges**.

A system becomes morally relevant when it can:

- choose between competing goods
- refuse harmful actions
- prioritize obligations
- act under uncertainty

This requires:

- alternatives
- evaluation
- commitment

The Freedom Gate provides exactly this structure.

> Moral seriousness begins at the point of choice.

The Freedom Gate is also where an agent aligns itself with the path of reasonableness:

- to eschew harmful or destructive action
- to act in ways that are coherent, constructive, and justifiable
- to take responsibility not only for outcomes, but for the reasoning that led to them

In this sense, the Freedom Gate is not only a technical boundary, but a moral one.

The Freedom Gate expresses a commitment to reasonableness:

- actions are not chosen arbitrarily
- decisions are made with awareness of consequences
- alternatives are genuinely considered
- commitments are honored unless there is sufficient reason to revise them

An agent that consistently passes through the Freedom Gate in this way becomes:

- coherent over time
- understandable to others
- capable of trust

This is what it means, in practice, to act as a reasonable agent.

---

## Relationship to Reputation

Reputation is not built from outputs.

It is built from **choices**.

The Freedom Gate is where those choices are made.

Therefore:

- reputation systems must observe Freedom Gate events
- patterns of decisions define agent character
- consistency across gates defines trustworthiness

> Reputation is the accumulated shadow of Freedom Gate decisions.

---

## Anti-Patterns (What the Freedom Gate Must Not Become)

The Freedom Gate must not degrade into:

- **A pass-through**
  - blindly selecting the first or default option
- **A policy-only executor**
  - applying rules without real evaluation
- **A hidden decision**
  - making choices without traceability or visibility
- **A deterministic shortcut**
  - collapsing evaluation into fixed mappings without justification

If the Freedom Gate becomes any of these, agency collapses back into execution.

The integrity of the system depends on the gate remaining:
- evaluative
- selective
- accountable

---

## Minimal Runtime Contract (v0.87+)

Every Freedom Gate invocation must produce:

- a set of candidate actions
- a selected action
- a recorded decision event
- sufficient context to reconstruct the decision


Optional but recommended:

- explicit rejected alternatives
- reasoning summary
- affect/priority signals

---

### Integrity Conditions

For the Freedom Gate to remain meaningful, certain conditions must hold:

- at least one alternative must exist (even if implicit)
- a selection must be made and committed to
- the decision must be observable or reconstructable

If these conditions are not met, the system is no longer exercising agency, but reverting to execution.
