# Five-Minute Sprint Console Demo

## Purpose

`WP-03` makes the C-SDLC visible as a governed creative-production process,
not only as a final artifact or an abstract packet family.

The demo wraps the real `Starharvest` HTML game sprint in one visual console so
other people can see:

- the compressed five-minute storyboard clock
- the named roles
- the work-state transitions
- the review and friction points
- the tracked artifacts
- the final launch and verdict boundary

## Current Proof Surface

`WP-03` is anchored by:

- `demos/v0.91.3/five_minute_sprint_console_demo.html`
- `demos/v0.91.3/five_minute_sprint_console_demo.css`
- `demos/v0.91.3/five_minute_sprint_console_demo.js`
- `adl/tools/demo_v0913_five_minute_sprint_console.sh`
- `docs/milestones/v0.91.3/review/five_minute_sprint_console/`

## What It Proves

- the C-SDLC mini-sprint can be explained as a governed issue wave instead of
  a mysterious one-shot output
- process, artifact, review, and launch truth can share one visual surface
- the Starharvest artifact can be presented as the result of a bounded creative
  production lane, not as a detached HTML file

## What It Does Not Prove

- literal measured five-minute end-to-end engineering success
- hidden scheduler automation
- unrestricted autonomous delivery

## Notes

This demo should stay disciplined about the clock:

- it is a compressed storyboard replay
- not a measured proof that the full wave completed in five real minutes
