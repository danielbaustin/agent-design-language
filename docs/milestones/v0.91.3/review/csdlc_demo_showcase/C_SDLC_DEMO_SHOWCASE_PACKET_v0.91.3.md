## Packet Identity

- packet name: `C-SDLC Demo Showcase`
- issue / WP: `demo WP-05 / #3224`

- purpose: package the merged demo mini-sprint evidence into one review surface

## Bounded Purpose

Make the merged mini-sprint evidence easy to inspect, share, and challenge
without claiming more than the current artifacts support.

## Recommended Demo Order

1. `Starharvest` HTML game
   - best first visual payoff
   - strongest public-facing artifact-to-effort ratio
2. Five-minute sprint console
   - explains the process that produced the visible artifact
   - keeps roles, review, and proof truth adjacent to the artifact
3. Podcast Studio v2
   - broadens the story from game artifact to recurring production system
   - shows deterministic packet generation and truthful audio boundaries
4. Five-minute-sprint first proof
   - closes with the governed process verdict
   - shows that bounded C-SDLC proof is real even though literal five-minute
     completion is still non-proving

## Strongest Claims

- ADL can present a bounded creative-production wave with visible roles,
  artifacts, review surfaces, and explicit non-claims.
- The mini-sprint now has three concrete front-stage demo lanes:
  `Starharvest`, sprint console, and Podcast Studio v2.
- The process verdict is disciplined: the governed first-proof story is real,
  while the literal five-minute target is still not proven.

## Non-Claims

- This package does not claim that arbitrary software can be built perfectly in
  five minutes.
- This package does not claim that the C-SDLC is fully automated or
  production-hardened.
- This package does not claim that the demos are publication-ready products.
- This package does not claim that the broader ADL strategy is settled by this
  mini-sprint.
