# Sprint Console Storyboard

## Compressed Replay Clock

The console uses a compressed `00:00 -> 05:00` storyboard clock.

This is a visual explanation of the mini-sprint flow, not measured end-to-end
engineering runtime.

## Replay Beats

1. `00:00` brief accepted and proof contract locked
2. `00:40` roles assigned
3. `01:30` mechanics and visual thesis stabilized
4. `02:25` implementation running
5. `03:15` playable Starharvest artifact opened
6. `03:55` QA review and packet surfaces land
7. `04:30` browser-capture friction remains explicit
8. `05:00` launch-ready bounded verdict

## Role Rail

- game designer
- mechanic designer
- art director
- frontend implementer
- QA reviewer
- release packager

## Work-State Board

The console must show:

- `planned`
- `running`
- `reviewed`
- `blocked`
- `complete`

without implying hidden automation or an invisible scheduler.

## Review And Friction

The console must keep at least one bounded friction surface visible:

- browser-capture proof remains partial in this environment unless later proof
  shows otherwise

That friction is important because it demonstrates governed truth instead of
victory-by-default storytelling.
