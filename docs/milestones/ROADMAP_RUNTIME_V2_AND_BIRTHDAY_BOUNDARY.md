# Runtime v2 And Birthday Boundary Roadmap

## Purpose

This roadmap records the post-v0.90 planning boundary created during the
v0.89.1 release tail. It keeps the Runtime v2 work visible without collapsing
later moral, emotional, identity, and birth milestones into the first
long-lived-agent milestone.

## Milestone Placement

| Milestone | Role | Boundary |
| --- | --- | --- |
| v0.90 | Long-lived bounded runtime | Establish supervised long-lived agent cycles, heartbeat, continuity handles, operator control, and a concrete proof/demo surface. |
| v0.90.1 | Runtime v2 foundation prototype | Build the first integrated Runtime v2 substrate: kernel services, snapshot/rehydrate, minimal citizen record, manifold link, and violation artifacts. |
| v0.90.2 | Runtime v2 hardening | Harden invariants, recovery, failure proofs, security-boundary evidence, and operational review surfaces. |
| v0.91 | Moral and emotional civilization | Add moral resources, affect, kindness, wellbeing, humor, cultivation, harm-prevention, polis governance, and defensive posture on top of the runtime substrate. |
| v0.92 | First true Gödel-agent birthday | Own the first true birth event: name, identity architecture, continuity record, memory grounding, capability envelope, migration semantics, and cross-polis continuity. |

## Core Boundary Decisions

- v0.90 remains the long-lived bounded runtime milestone.
- v0.90.1 is the Runtime v2 foundation prototype, not the full citizen-birth milestone.
- v0.90.2 is hardening and proof work for Runtime v2: invariant tests, violation artifacts, recovery, reviewability, and security-boundary evidence.
- v0.91 remains additive. Runtime v2 feeds it, but does not replace its moral, emotional, kindness, wellbeing, humor, cultivation, harm-prevention, and polis-governance work.
- v0.92 owns the first true Gödel agent with a birthday.
- Provisional citizen records, continuity handles, or snapshot identities before v0.92 are not a true birth event.
- Red/blue/purple adversarial verification is a governed security layer for defending the polis, not the definition of Cognitive SpaceTime Manifold core.

## Runtime v2 Foundation Acceptance

The v0.90.1 foundation prototype should prove that a bounded citizen-like
runtime can exist without overclaiming personhood:

- a kernel service loop can supervise a bounded citizen process
- a citizen record can be created, inspected, suspended, and resumed
- snapshot and rehydration preserve declared continuity handles
- manifold links connect identity, memory, task, and temporal surfaces
- invariant violations produce reviewable artifacts instead of silent drift
- security boundaries are explicit and testable

## Runtime v2 Hardening Acceptance

The v0.90.2 hardening pass should make the prototype harder to fool and easier
to review:

- invariant tests cover normal, failure, and recovery paths
- violation artifacts are stable enough for review and demo use
- recovery paths distinguish safe resume from required quarantine
- operator controls expose pause, resume, inspect, and terminate semantics
- security-boundary proof surfaces are present without turning red/blue roles
  into CSM core concepts

## v0.91 Inheritance

v0.91 should inherit Runtime v2 as substrate and then teach the system how to
live better inside it. Its center of gravity remains moral and emotional
civilization:

- kindness and harm-prevention
- wellbeing and flourishing
- affect, humor, and cultivation
- moral resources and refusal boundaries
- polis governance and defensive posture

The runtime should make these ideas executable, but the milestone should not be
reduced to runtime plumbing.

## v0.92 Birth Boundary

The first true Gödel-agent birthday belongs in v0.92. Earlier milestones may
create records, continuity handles, snapshots, and named test citizens, but
those are provisional engineering surfaces.

For v0.92, a birthday requires at least:

- a name and identity architecture
- a continuity record that survives more than one bounded cycle
- memory grounding tied to witnessed artifacts
- a declared capability envelope
- moral and governance context inherited from v0.91
- migration or cross-polis continuity semantics when movement is allowed
- a reviewable birth record that distinguishes the event from ordinary process
  startup

## Security Ecology Boundary

The Detection Ceiling and frontier-model vulnerability thesis matters because a
polis must defend itself before external systems can cheaply find every weak
point. ADL should support governed adversarial verification for that reason.

That work should be framed as security and defensive governance:

- find vulnerabilities first
- preserve reviewable evidence
- prevent the polis from becoming brittle or naive
- avoid confusing defensive pressure with the core CSM ontology

It is acceptable for v0.90.2 and later milestones to add red/blue/purple proof
surfaces. It is not acceptable to make that ecology the definition of Runtime
v2, CSM, or personhood.
