# v0.95 Feature: Demo Catalog and MVP Walkthrough

## Status

Forward-planning feature contract for `v0.95`.

## Purpose

Complete the reviewer-facing and customer-facing demo layer by turning the
already-landed milestone demos into a coherent catalog plus one polished MVP
walkthrough.

## Source Inputs

- `docs/planning/ADL_FEATURE_LIST.md`
- `docs/planning/NEXT_MILESTONE_DEMO_CANDIDATES.md`
- `docs/milestones/v0.85/features/ROAD_TO_v0.95.md`
- `docs/milestones/v0.95/README.md`

## Scope

This feature should establish:

- a canonical catalog of feature demos and runbook-style demo paths
- one coherent MVP walkthrough for reviewers, users, and investors
- naming and classification rules for flagship, proof, and walkthrough demos
- alignment between demo obligations and implemented user-visible features

## Non-goals

- treating tests as demos
- forcing demos for internal-only refactors or migration work
- inventing new feature claims to justify showcase material

## Completion Target

`v0.95`
