# Release Plan - v0.90.3

## Release Intent

v0.90.3 should release only when the citizen-state substrate proof is coherent,
reviewed, and bounded.

The release should show that ADL can protect continuity-bearing citizen state
through local-first sealed checkpoints, signed envelopes, append-only lineage,
witnesses, receipts, redacted projections, standing/access boundaries,
sanctuary/quarantine behavior, and challenge/appeal evidence.

## Release Evidence

Required before release:

- inheritance and unsafe-assumption audit
- private-state format decision and fixture
- signed envelope and trust-root fixture
- local sealed checkpoint fixture or explicit local sealing boundary
- append-only lineage ledger proof
- continuity witness and receipt fixtures
- anti-equivocation negative test
- sanctuary/quarantine proof
- redacted Observatory projection and leakage scan
- standing, communication, and access-control proof
- integrated citizen-state demo packet
- challenge/appeal proof
- threat model
- economics placement decision
- internal and external review records

## Review Tail

The release tail should follow the v0.87.1/v0.90.2 pattern:

1. docs, quality, and review convergence
2. internal review
3. external / third-party review
4. accepted finding remediation or explicit deferral
5. release notes and release evidence
6. release ceremony

## Release Non-Claims

- no first true Gödel-agent birthday
- no full moral/emotional civilization
- no complete migration or cross-polis continuity
- no full economics or contract-market substrate
- no cloud enclave deployment requirement
- no unrestricted operator inspection

## Handoff

v0.90.3 should hand off:

- v0.90.4 full economics and contract-market substrate
- v0.91 moral/emotional civilization layer
- v0.92 first birthday and identity/capability rebinding
- later confidential-computing backend proof if local sealing needs hardware
  attestation
