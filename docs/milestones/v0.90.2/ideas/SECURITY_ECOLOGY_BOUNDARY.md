# Security Ecology Boundary

The frontier-model vulnerability thesis matters: if stronger models can find
weaknesses quickly, the polis needs governed defensive verification.

v0.90.2 may include bounded adversarial-style checks where they defend the
polis. It must not make red, blue, or purple roles the definition of Runtime v2,
CSM, or personhood.

Good framing:

- find vulnerabilities first
- preserve reviewable evidence
- protect the polis
- keep security ontology separate from Runtime v2 core ontology
