# Structured Prompt Records

This directory is the tracked home for public structured prompt records.

ADL uses these tracked records to preserve an auditable history of:

- Structured Task Prompts
- Structured Implementation Prompts
- Structured Output Records

The `.adl/` workspace remains useful for local drafting, generated intermediates, and editor scratch state, but it is not the canonical public record.

Official lifecycle transitions should promote durable prompt artifacts into tracked records before they become authoritative workflow state.
