ADL Task Bundle Editor

A bounded task-bundle workspace shell for Structured Task Prompts, Structured Implementation Prompts, and a real review-first Structured Output Record surface. This editor keeps the current local task-bundle model visible while reducing fragile raw markdown editing.

ADL Language Contract

This editor does not author full ADL documents, but it should still describe the same canonical language shape as the runtime, schema, and docs.

          Current Boundaries
          
      
            
        
          • This issue refreshes the task-bundle workspace shell for the current ADL workflow skill family.
            • 
        
          • The editor consumes the canonical ADL six-primitives contract: providers, tools, agents, tasks, workflows, and singular run.
            • 
        
          • Patterns and signature are top-level language features, not extra primitives.
            • 
        
          • STP and SIP remain editable authoring surfaces, and SOR remains an editable review-first closeout surface.
            • 
        
          • SOR now has a bounded review-first surface for evidence, integration state, and follow-up structure.
            • 
        
          • The review flow now turns SOR evidence, proof, and follow-up fields into a reviewer-facing checklist and bounded recommendation.
            • 
        
          • The editor prepares current lifecycle commands for a human to copy and run; browser direct execution remains out of scope.
            • 
        
          • The current canonical workflow state lives under .adl/; older docs/records/ examples are historical proof surfaces only.
